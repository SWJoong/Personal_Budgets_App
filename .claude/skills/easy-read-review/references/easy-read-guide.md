# 쉬운 글(Easy Korean) 작성 원칙

**주 출처**: 한국장애인고용공단 「알기 쉬운 자료 제작 안내서」(2021, 고용개발 2021-04)
**보조 출처**: 보건복지부·한국지적발달장애인복지협회 「발달장애인을 위한 쉽게 접근할 수 있는 정보 만들기」(2017 개정판)

---

## 1. 쉬운 정보의 두 단계

| 단계 | 대상 문해 수준 | 초점 |
|------|--------------|------|
| **보통 읽기 (Plain Language)** | 수준 3 (중학교 수준) | 어휘·문장 단순화, 텍스트 중심 |
| **쉬운 글 (Easy Korean)** | 수준 1–2 (초등 수준) | 어휘·문장 + 레이아웃 + 이미지 모두 |

> 개인예산제 앱은 **쉬운 글** 기준을 목표로 한다.

### 1-1. 쉬운 정보의 기본 원칙 (2017 안내서)

1. 당사자가 정보를 주고받는 데 **선호하는 방법**을 찾아야 한다.
2. 당사자의 **이전 경험**을 고려해야 한다.
3. 당사자가 **어떤 정보를 필요로 하는지, 왜 필요한지** 정확하게 이해해야 한다.
4. 당사자가 **언제** 그 정보를 필요로 하는지를 이해해야 한다.
5. 정보를 주고받을 수 있는 **가장 적합한 사람**이 누군지 결정해야 한다.
6. 정보가 **어떻게 주고받아져야 하는지**를 이해해야 한다.

> '얼마만큼의 정보를 주느냐'가 아니라, '사람들이 그것을 얼마나 이해했느냐'가 더 중요하다.

### 1-2. 발달장애인 특성 고려사항

- 인칭 대명사 사용에 어려움이 있는 경우가 많다.
- 가상인물, 비유적·은유적 표현, 가상놀이 등에 제한이 있다.
- '그', '그녀', '영수' 등 3인칭 대명사와 가상 인물이름을 본인으로 연결하여 이해하는 데 제한이 있다.
- 청각적·언어적 정보보다 **시각적 정보에 더 민감하게 반응**하는 경향이 있다.
- 어려운 한자어, 외국어 등에 이해가 어렵다.

## 2. 쉬운 정보 개발 순서 (7단계)

| 단계 | 작업 | 설명 |
|------|------|------|
| 1단계 | 기본 문서 만들기 | 일반적인 작성 방식으로 문서 작성. 가능한 요약 형식 권장 |
| 2단계 | 주요 단어 선택하기 | 문서를 읽고 각 문장의 핵심 단어 선택 |
| 3단계 | 간단한 문장 만들기 | 짧은 단락이나 간단한 문장으로 변환. 이 단계에서 당사자 자문 가능 |
| 4단계 | 그림 고르기 | 문장이나 단어에 맞는 그림(이미지) 선택 |
| 5단계 | 문서 초안 완성 | 그림과 문장을 배치하여 초안 완성 |
| 6단계 | 검토받기 | 발달장애인(당사자)을 통해 문서 초안에 대해 자문 받기 |
| 7단계 | 완성하기 | 자문 내용을 반영하여 최종 문서 완성 |

> ※ '보통 읽기' → '쉬운 글' 순서로 수정하며, 각 단계마다 당사자 검증을 받는다.

## 3. 어휘 원칙

### 3-1. 단어 선택
- ✅ 짧고 단순한 단어를 선택한다
- ✅ 어려운 단어가 꼭 필요하면 **굵게 + 파란색** 표시 후 괄호 없이 별도 설명 추가
- ✅ 중요한 단어는 **굵게(볼드)** 처리
- ✅ 같은 내용은 항상 같은 단어로 일관성 유지
- ✅ 복잡하고 어려운 단어 대신 더 쉽고 더 짧은 단어가 있는지 항상 확인
- ❌ 한자어 사용 금지 (예: 지원 → 돕다, 집행 → 쓰다, 과장 → 사실보다 크게 말하다)
- ❌ 외래어·약어 금지 (예: 바우처 → 이용권, 서비스 → 도움, 로그인 → 들어가기)
- ❌ 접속사 금지 (그리고, 따라서, 그러므로)
- ❌ 속어·은어·약어·방언·속담·고사성어 금지
- ❌ 추상명사 최소화 (예: 활용 → 사용, 지속 → 계속)
- ❌ 단축형 표현 금지 (예: 내돈내산)

### 3-2. 독자·작성자 지정
- ✅ 독자는 **'나'** 또는 **'당신'** 으로 지칭
- ✅ 기관·앱은 **'우리'** 로 지칭
- ✅ 독자와 작성자를 밝힌 후에는 주어 없이 작성할 수 있다
- ❌ 3인칭 대명사('그', '그녀', 가상인물) 사용 금지
  - 이유: 발달장애인은 가상인물을 본인 이야기로 연결하기 어려움

### 3-3. 숫자 표기
- ✅ 아라비아 숫자 사용 (세 → 3, 열두 → 12)
- ✅ 숫자 앞뒤 한 칸 띄어쓰기
- ✅ 시간: 12시간 표기제 (오전 9시, 오후 3시)
- ✅ 날짜: 년, 월, 일 (2024년 3월 15일)
- ✅ 금액: 아라비아 숫자 + 돈 단위 (50,000 원)
- ✅ 전화번호: 국번 숫자를 따로 묶어서 표기 (02 123 4567)

### 3-4. 전문용어 처리
- 전문용어는 특정 분야 전문가들이 주로 사용하는 단어로, 해당 분야를 모르는 사람에게는 이해하기 어렵다.
- 꼭 필요한 전문용어는 다음 형식으로 제시:
  ```
  **예산** 이란 내가 쓸 수 있는 돈입니다.
  ```
- 법률 용어·고유명사(기관명, 지명 등)는 그대로 사용하되, **굵게 + 파란색** + 단어 사전에 쉽게 풀이하여 설명
- 대체할 수 있는 쉬운 단어가 없다면, 단어의 의미를 간단하게 설명해야 한다.

## 4. 문장 원칙

### 4-1. 문장 구조
- ✅ 1문장 = 1아이디어 = 1서술어
- ✅ 짧고 간단한 문장 (5–8단어 권장, 이미지 사용 시)
- ✅ 이미지 없을 때: 문장당 5–10단어
- ✅ 능동태 사용 (예: 처리됩니다 → 우리가 처리합니다)
- ✅ 긍정문 사용 (예: 입력하지 않으면 → 꼭 입력해야 합니다)
- ✅ 구어체 사용 (예: 선택을 함 → 선택합니다)
- ✅ 주어+동사+목적어를 최대한 붙여서 사용 (형용사·부사 제외)
- ❌ 수동태 금지
- ❌ 부정문 지양
- ❌ 개조식 문장(~함, ~임) 금지
- ❌ 이중 부정 금지 (예: 불가능하지 않다)
- ❌ 한 페이지에서 다음 페이지로 문장을 이어가지 않는다

### 4-2. 삭제할 요소
- ❌ 부사·형용사 최소화 (빨리, 최대한, 가능한, 곧, 즉시, 가까운, 비슷한 등)
- ❌ 대명사(그것, 이것, 저것) 대신 명사 사용
- ❌ 특수문자·부호 금지 (괄호 (), 하이픈 -, 슬래시 /, 밑줄 _, #, &, ~, %, ^, ₩)
- ❌ 동사의 명사화 최소화 (예: 향후 원격수업을 수업에 활용할 것입니다 → 앞으로 원격수업을 활용합니다)

### 4-3. 정보 구조화
- ✅ 문장 분류나 정보 구성을 위해 글머리 기호(●)나 머리글을 사용
- ✅ 글머리 기호(●)는 한 문단 내에서 최대 4개까지
- ✅ 글머리 기호 사용 시, 문장의 가장 마지막에만 마침표 사용
- ✅ 불필요한 정보를 제거하고 요약정리
- ✅ 복잡한 정보를 단순화하고 일상생활의 예를 사용하여 설명
- ✅ 유사한 정보끼리 모으기

## 5. 레이아웃 원칙 (앱 UI 적용 기준)

### 5-1. 텍스트 밀도
- 이미지(아이콘) 있을 때: 1문장 = 5–8단어
- 이미지 없을 때: 1문장 = 5–10단어

### 5-2. 글꼴
- 고딕·명조 기본 글꼴 권장 (맑은 고딕, 나눔 고딕, 굴림, 돋움체, 신명조 등)
- 본문 최소 14pt / 제목 16pt 이상
- 자간(0), 장평(100%)은 기본으로 사용
- 이탤릭·밑줄·따옴표·기울이기·편지체 금지

### 5-3. 정렬
- 왼쪽 정렬 (가운데·오른쪽 정렬 지양)
- 텍스트 여러 열 배치 금지
- 줄바꿈은 단어별로 수행

### 5-4. 여백 및 줄 간격
- 전체 페이지는 최소 2.5cm 너비의 넓은 여백
- 이미지 사용 시 왼쪽 여백 5cm (또는 최소 8cm 그림 공간 확보)
- 문단 내 줄 간격: 이중 줄 간격(220% 또는 1.3 line 이상)
- 문단 사이: 4줄 간격
- 문장 사이: 충분한 간격 확보

### 5-5. 문서 배경
- 흰색 페이지에 검은색 텍스트가 기본
- 필요 시 고대비 색상 활용 (예: 검정 바탕에 노란색 글씨)
- 밝은 색상(형광색, 유아틱한 배경) 과도 사용 금지
- 한 문서에 너무 다양한 색상 금지
- 파스텔 음영은 흰 종이보다 눈부심을 줄여줄 수 있음
- 난독증이 있는 사람에게는 노란색 배경이 도움

### 5-6. 문서 구성
- 짧고 간단한 제목을 사용하며, 제목은 문서의 정보를 소개
- 페이지 번호를 항상 사용 (기본 텍스트와 동일한 글꼴·크기)
- 전체 문서는 20쪽 이내 권장
- 한 문장이 두 줄로 나뉘면 두 번째 줄에서 문장을 완성
- 한 페이지 내에서 문장을 시작하고 마무리

### 5-7. 버튼 레이블 특칙 (앱 UI)
- 동사 + 목적어 형식 권장 (예: "저장하기", "예산 확인하기")
- 추상적 단어 지양 (예: "제출" → "보내기", "확인" → "맞습니다")
- 1버튼 = 1동작

### 5-8. 에러 메시지 특칙 (앱 UI)
- 문제가 무엇인지 + 어떻게 하면 되는지를 함께 알려준다
- 예: "오류가 발생했습니다" → "저장이 안 되었습니다. 다시 눌러 주세요."
- 기술적 용어(서버 오류, 404, timeout) 사용 금지
- 사용자를 탓하는 표현 금지

## 6. 이미지(아이콘·그림) 원칙

### 6-1. 기본 원칙
- ✅ 아이콘·그림은 텍스트와 함께 사용 (아이콘만 단독 사용 금지)
- ✅ 이미지를 본 다음 텍스트를 읽도록 배치 (왼쪽 → 오른쪽 읽기)
- ✅ 아이콘·이미지 왼쪽 정렬 (텍스트 왼쪽에 배치)
- ✅ 이미지 주위에 여백 확보 (텍스트 기준 0.5cm 이상)
- ✅ 이미지는 가급적 크게
- ✅ 가능하면 컬러 이미지 사용
- ✅ 단순하고 단색 배경 사용
- ✅ 문서 전체에 걸쳐 동일한 콘셉트·동일한 이미지 스타일 유지

### 6-2. 이미지의 역할 (2017 안내서)
- 문장을 다 읽지 않아도 어떤 내용인지 이해하는 데 도움
- 글을 읽지 못하는 사람에게 내용의 요점을 명확하게 전달
- 문장 이해력이 부족한 사람들에게도 도움
- 문장을 덜 두려워하게 만들고 문서가 더 친근하게 느껴짐
- 한 지면에 담을 수 있는 정보의 양을 제한하는 역할

### 6-3. 이미지 명료성
- ✅ 이미지가 실제 사물이나 활동과 유사한지 확인
- ✅ 이미지로 요점·주요 내용을 설명
- ✅ 전경과 배경이 혼동되지 않도록 확인
- ❌ 이미지의 배경이 산만하면 집중할 수 없음

### 6-4. 이미지 관련성
- 독자의 연령에 적합한 이미지 사용 (유치한 만화 지양)
- 독자의 문화를 고려 (예: 한국 문화에 맞는 이미지)
- 독자의 일상 삶 속에서 보편적으로 사용되는 이미지 활용

### 6-5. 추상 개념 표현
- ✅ 부정적인 개념: ✕ 표시 사용
- ✅ 특정 개념 강조: ○ 또는 색 변경 (화살표 대신)
- ❌ 화살표(←)·말풍선 사용 지양 — 시각적으로 따라가며 이해하기 어려움

### 6-6. 피해야 할 이미지
- ❌ 이미지에 글자 합성 금지 (이미지와 글자 혼용 최소화)
- ❌ 표·지도·도표 사용 금지 — 이해하기 어려울 수 있음
- ❌ 그래프·기호·말풍선·워터마크·추상적 표시 금지
- ❌ 단순히 장식용으로 이미지 삽입 금지
- ❌ 모든 단어에 이미지 사용 금지
- ❌ 뒤에 워터마크 텍스트나 그래픽 사용 금지

### 6-7. 시각적 도구 종류 선택 (2017 안내서)

| 종류 | 사용 시기 |
|:---|:---|
| **그림(삽화)** | 당사자가 경험한 것이 아닌 경우, 일반적인 상황이나 개념을 나타낼 때 |
| **사진** | 당사자가 경험한 것, 특정 사물·사람·장소를 설명할 때 |
| **심볼(상징)** | 키워드나 주요 생각 표현. 화장실·식사시간·금연 등 일반적이고 명백한 심볼에만 사용 |
| **픽토그램·로고·선화·일러스트** | 필요에 따라 적절히 선택. 저작권 확인 필수 |

> 사진은 한꺼번에 많은 정보를 제시하면 이해하기 어려울 수 있으므로, 특정 참조용으로만 사용한다.

### 6-8. 그림으로 설명하기 좋은 내용
- 특정 행동 (예: 예약하는 방법)
- 특정 프로세스 (예: 절차 중 발생할 수 있는 일)
- 해도 되는 것과 하지 말아야 할 것
- 특정 사건, 장소, 사람의 목록

### 6-9. 그림만으로 표현이 어려운 내용
- 사고(생각), 논쟁·토론이 필요한 내용
- 개념이나 추상적 주제 (예: 정체성, 시민정신)
- 통계 (예: 여론조사 결과 분석)

## 7. 당사자 참여 및 검증

### 7-1. 당사자 검증의 중요성
- 발달장애인은 쉽게 접근할 수 있는 정보에 관해서는 **전문가**이다.
- 자료를 검증받고자 할 때는 **다양한 당사자 검증가 그룹을 구성**해야 한다.
- 한 명의 당사자가 한 문서를 지속해서 검증하는 것은 **지양** — 반복학습 효과 발생 가능

### 7-2. 당사자 그룹 구분

| 당사자 참조 그룹 | 당사자 자문 그룹 |
|:---|:---|
| 기획 단계~최종 점검까지 모든 과정에 지속 참여 | 개발된 자료를 일회성 검증 |
| 쉬운 정보 개발 과정 중 지속적 역할 | 주로 최종 개발물에 대한 검증 담당 |

### 7-3. 당사자 검증 항목

| 검증 항목 | 확인 질문 |
|:---|:---|
| 레이아웃 및 표시 | 문서가 보기 쉬운가? |
| 사용 글 | 내용이 명확하고 이해하기 쉬운가? |
| 사용된 이미지 | 이미지가 의미가 있고 글을 이해하는 것을 도와주는가? |
| 용이성과 가독성 | 자료를 읽고 난 후 당사자가 정보를 이해하였는가? |

자료를 읽고 난 후 당사자가 답할 수 있어야 하는 것:
- 무슨 내용인가?
- 무엇을 해야 하는가?
- 어떤 것을 할 수 있는가?
- 어떻게 해야 하는가?

## 8. 법적 근거

- **유엔 장애인 권리협약 제21조** (2007): 정보와 의사표현의 자유, 접근 가능한 형식으로 제공 의무
- **발달장애인 권리보장 및 지원에 관한 법률 제10조** (2015): 국가·지방자치단체는 발달장애인이 이해하기 쉬운 형태로 정책정보 작성·배포 의무
- **장애인차별금지및권리구제에관한법률 제7조**: 장애인의 자기결정권 및 정보 접근 선택권 보장
- **발달장애인법 시행령 제4조**: 정책정보의 작성 및 배포에 포함될 사항 규정
- **영국 장애인차별금지법(DDA, 1995)**: 서비스 제공 시 장애인 접근 가능한 정보 제공 의무
